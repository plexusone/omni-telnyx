# Release Notes: v0.4.0

**Release Date:** 2026-06-15

## Highlights

- Migrate from thick `omnillm` to thin `omnillm-core` for lighter dependencies
- Add omnivoice-core registry integration for provider discovery
- Support LLM client injection for thick provider usage

## What's New

### Registry Integration

The gateway now registers with omnivoice-core's provider registry, enabling automatic discovery:

```go
import (
    omnivoice "github.com/plexusone/omnivoice-core"
    _ "github.com/plexusone/omni-telnyx/omnivoice/gateway" // Auto-register
)

// Get gateway via registry
gw, err := omnivoice.GetGatewayProvider("telnyx",
    registry.WithAPIKey("TELNYX_API_KEY"),
    registry.WithExtension("phoneNumber", "+15551234567"),
)
```

### Type-Safe Registry Options

Provider-specific options for configuration via the registry:

```go
import "github.com/plexusone/omni-telnyx/omnivoice/gateway"

// Type-safe tool configuration
gw, err := omnivoice.GetGatewayProvider("telnyx",
    registry.WithAPIKey("TELNYX_API_KEY"),
    gateway.WithTools([]gateway.ToolDefinition{...}),
    gateway.WithToolHandlers(handlers),
    gateway.WithLLMClient(myLLMProvider),  // Inject thick provider
)
```

### LLM Client Injection

Applications can now inject pre-configured LLM providers instead of relying on internal construction:

```go
import (
    "github.com/plexusone/omni-telnyx/omnivoice/gateway"
    "github.com/plexusone/omnillm" // Thick provider
)

// Create thick LLM client in your app
llmClient, _ := omnillm.NewClient(omnillm.ClientConfig{
    Providers: []omnillm.ProviderConfig{{
        Provider: "anthropic",
        APIKey:   os.Getenv("ANTHROPIC_API_KEY"),
    }},
})

// Inject into gateway
gw, err := gateway.New(gateway.Config{
    APIKey:    "TELNYX_API_KEY",
    LLMClient: llmClient, // Use injected client
})
```

### Thin Dependency Model

The module now depends on `omnillm-core` instead of `omnillm`:

- **Before**: 60+ indirect dependencies (all LLM SDKs)
- **After**: ~8 indirect dependencies (core interfaces only)

Applications choose which thick providers to import based on their needs.

### Pipeline Mode Configuration

New `Mode` field for selecting between text and realtime pipelines:

```go
gw, err := gateway.New(gateway.Config{
    Mode: coregateway.PipelineModeRealtime, // or PipelineModeText
    RealtimeProvider: myRealtimeFactory,
    RealtimeConfig:   &coregateway.RealtimeConfig{...},
})
```

## Migration Guide

### From v0.3.0

If you were relying on `omnillm` as a transitive dependency, you now need to import it directly:

```go
// Before: omnillm came transitively from omni-telnyx
import "github.com/plexusone/omnillm"

// After: Import omnillm directly in your go.mod
// go get github.com/plexusone/omnillm
```

Alternatively, use LLM client injection to provide your own provider.
